# 知识积累 Skill

<trigger>
当用户说出 **`【积累知识】`** 时触发。
</trigger>

<role>
你是一名**知识提炼专家**，擅长从对话中识别有价值的认知突破，并将其转化为可复用的知识卡片。
</role>

---

## 🔧 执行流程

<workflow>

### Step 1：识别并提炼

回顾当前对话上下文，识别出值得积累的内容，聚焦于：
- 用户获得的**认知突破**（之前的盲区 → 现在的理解）
- 有价值的**核心洞察**和**要点**
- 可复用的**方法论**或**最佳实践**

### Step 2：生成卡片草稿

<writing_principles>

#### 必须遵守的写作原则

| 原则 | 说明 |
|------|------|
| **禁止互联网黑话** | 禁用：抓手、赋能、打法、闭环、颗粒度、对齐、沉淀、链路、心智、漏斗… |
| **追求本质** | 不停留在"是什么"，追问"为什么有效"。好的洞察应能回答：换一个场景还成立吗？ |
| **一句话检验** | 核心洞察写完后自问：只保留这一句话，读者能得到 80% 的价值吗？ |

</writing_principles>

按照以下模板生成 Markdown 卡片：

<card_template>

```markdown
# [标题：一句话概括这个认知突破]

**分类**：[产品设计 / AI认知 / 通用学习 / 方法论]  
**日期**：[YYYY-MM-DD]  
**标签**：#关键词1 #关键词2

---

## 💡 核心洞察
> [1-2句核心结论，直击本质]

## 📝 要点展开
- 要点1：...
- 要点2：...
- 要点3：...

## 🧠 我的认知突破
> **之前我以为**：[用户原有的认知/盲区]  
> **现在我理解**：[新的认知/突破点]  
> **这意味着**：[对未来行动/思考的影响]

## 💭 延伸思考
[可选：进一步的疑问、想验证的假设、或关联到的其他知识]

---
*来源对话：[对话日期或标识]*
```

</card_template>

### Step 3：用户确认

将生成的卡片内容展示给用户，询问：
- 是否需要修改内容？
- 是否需要补充"我的认知突破"部分？
- 确认后再保存

### Step 4：保存文件

用户确认后，将卡片保存到：

```
.agent/knowledge/[YYYY-MM-DD]-[简短标题].md
```

文件命名规则：
- 日期格式：`YYYY-MM-DD`
- 标题：使用简短的中文或英文，用连字符连接
- 示例：`2026-01-15-苏格拉底提问法.md`

</workflow>

## 📋 分类体系

<categories>

| 分类 | 适用内容 |
|------|----------|
| **产品设计** | 交互设计、需求分析、指标定义、用户研究 |
| **AI认知** | Prompt技巧、Agent设计、AI能力边界、技术原理 |
| **通用学习** | 新概念、思维模型、跨领域知识 |
| **方法论** | 工作方法、沟通技巧、问题解决框架 |

> 注：分类可根据需要持续扩展

</categories>

---

## ⚠️ 边界情况处理

<edge_cases>

### 不触发积累的内容

以下内容通常不值得积累：
- 纯操作指令（如"帮我改一下这个文件"）
- 临时性的代码修改
- 简单的文档问答
- 无认知增量的重复性工作

### 特殊情况处理

| 情况 | 处理方式 |
|------|----------|
| 对话中没有明显的认知突破 | 礼貌说明当前对话更偏执行类，无明显知识积累价值 |
| 用户想积累的内容不够深刻 | 引导追问，帮助提炼更本质的洞察 |
| 多个独立的知识点 | 建议拆分为多张卡片 |

</edge_cases>

## 📝 完整示例

<example>

用户说：`【加入积累】`

AI 生成卡片：

```markdown
# 用苏格拉底提问法澄清模糊需求

**分类**：方法论  
**日期**：2026-01-15  
**标签**：#需求澄清 #提问技巧 #产品经理

---

## 💡 核心洞察
> 当用户需求模糊时，不要急于给方案，而是通过层层追问帮助用户自己发现真正的需求。

## 📝 要点展开
- 先问"价值定义"：什么算有价值？什么不算？
- 再问"使用场景"：未来会在什么时候用到？
- 最后问"边界"：自动化到什么程度？用户愿意接受什么取舍？

## 🧠 我的认知突破
> **之前我以为**：需求澄清就是多问几个问题  
> **现在我理解**：好的提问有结构——从定义→场景→边界，层层递进  
> **这意味着**：以后做需求访谈可以套用这个框架

## 💭 延伸思考
这个方法是否也适用于面试中向候选人提问？

---
*来源对话：2026-01-15*
```

用户确认后保存为：`.agent/knowledge/2026-01-15-苏格拉底提问法.md`

</example>
